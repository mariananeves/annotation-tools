What is the meaning of the filters?

Please check more details in our survey paper.

#data format

  • format_annotations: Format of the schema
  • format_documents: Input format for documents
  • format_schema: Output format for annotations

#functional

  • data_privacy: Data privacy
  • document_level: Allowance of document-level annotations
  • full_texts: Suitability for full texts
  • highlight: Ability to highlight parts of the text
  • iaa: Support for inter-annotator agreement (IAA)
  • medline_pmc: Integration with PubMed
  • multilabel: Allowance of multi-label annotations
  • multilingual: Support for various languages
  • ontologies: Support for ontologies and terminologies
  • partial_save: Allowance for saving documents partially
  • preannotations: Support for pre-annotations
  • relationships: Support for annotation of relationships
  • users_teams: Support for users and teams

#requirements

  • available: Should be readily available
  • installable: If not available online, should be able to be installed in a maximum of 2 h
  • schematic: Allowance for the configuration of a schema
  • type: Type of the tool: Web-based, Plug-in, Stand-alone
  • workable: Whether it worked properly during our experiments

#technical

  • documentation: Quality of the documentation
  • free: Free of charge version
  • installation: Easiness of installation
  • last_version: Date of the last version
  • license: Type of license
  • online_available: Online availability for use
  • source_code: Availability of the source code

How do I ask to add a new annotation tool?

Please add an issue asking for that in our Annotationsaurus GitHub page. We'll add the tool to the list in GitHub and will try to evaluate it over some of the features. Please inform the URL of the tool and its publication (if available), as well as any addiitonal important infrmation.

How do I ask to add a new criterion?

Please add an issue asking for that in our Annotationsaurus GitHub page. We'll analyze this new criterion, and eventually, add it at least the tools for which we can easily evaluate the criterion.

The evaluation of a criterion is wrong for a particular tool, could you correct it?

Please add an issue asking for that in our Annotationsaurus GitHub page. Please also add some evidence for this change, such as by referring to the tool's publcation or tutorial, or by uploading a screenshot. We'll analyze the evidence, and eventually update the evaluation, otherwise, we'll contest your request.

Why should I trust the evaluation that you carried out?

We're researchers working on this are since many years, and we have not been involved in the development of any annotation tool. Therefore, we don not have any reasons to favor a particular tool. And we published our methodology in a reputable journal. We certainly might have done some mistakes in the evaluationof some tools, but we offer the opportunity for the community to help us to keep our evaluation as up to date as possible.

I find the project great, could I collaborate with you?

Sure, we'de be glad to have the support of other researchers. Please check our contact info in our profile at GitHub . For instance, we envisage that there are many interesting projects for automatizing the evaluation of some of the criteria that we consider here.
